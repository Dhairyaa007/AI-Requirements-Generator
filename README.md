# AI Requirements Engineering Assistant
An AI-powered business analysis application that transforms natural language business requirements into structured, consulting-grade software documentation. The application leverages large language models (LLMs) to automate key requirements engineering activities, enabling faster analysis, improved documentation quality, and more consistent project deliverables.

## Live Demo

## Project Overview
The AI Requirements Engineering Assistant streamlines the early stages of the Software Development Life Cycle (SDLC) by converting business problem statements into structured requirements documentation. Designed for business analysts, product managers, project teams, and consultants, the application generates standardized outputs that support requirements analysis, solution design, and project planning.

The application automatically produces:
* Agile User Stories
* Acceptance Criteria
* Functional Requirements
* Positive and Negative Test Cases
* Edge Cases
* Risks and Assumptions
* Project Dependencies
* AI and Automation Opportunities
* Requirement Quality Assessment (Clarity, Completeness, and Risk)
* Downloadable PDF Consulting Report

The project demonstrates how generative AI can be integrated into business analysis workflows to improve efficiency, consistency, and documentation quality while supporting AI-enabled digital transformation initiatives.

## Key Features
* Automated business requirement analysis using large language models
* Structured generation of software requirements and supporting documentation
* Industry-aware analysis for domains such as Financial Services, Healthcare, Retail, and the Public Sector
* Requirement quality assessment based on clarity, completeness, and implementation risk
* Automated generation of consulting-style PDF reports
* Interactive web interface built with Streamlit

## AI Model
The application uses the **LLaMA 3.3** model served through the **Groq API** to generate structured business analysis artifacts. Prompt engineering techniques are used to guide the model toward consistent, consulting-oriented outputs suitable for requirements engineering and software delivery.
